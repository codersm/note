# 优化方法
## 1、子查询优化法
先找出第一条数据，然后大于等于这条数据的id就是要获取的数据。
缺点：数据必须是连续的，可以说不能有where条件，where条件会筛选数据，导致数据失去连续性。

## 2、倒排表优化法
倒排表法类似建立索引，用一张表来维护页数，然后通过高效的连接得到数据。
缺点：只适合数据数固定的情况，数据不能删除，维护页表困难。
